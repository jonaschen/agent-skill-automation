## Cross-Cutting Analysis

### Managed Agent Platforms Converge
Both vendors now offer managed agent hosting: Anthropic's **Managed Agents** (public beta) and Google's **Vertex AI Agent Engine** (GA). Key comparison:
- **Managed Agents**: Container-based, built-in tools (Bash, file ops, web, MCP), SSE streaming, 60 creates/min. Research preview: multiagent, outcomes, memory.
- **Agent Engine**: Cloud-native, ADK integration, Sessions/Memory Bank/Code Execution GA, observability dashboard, self-healing plugins, private VPC, HIPAA support, session rewind.
- **Gap**: Managed Agents launched later but includes native MCP server support. Agent Engine has more enterprise features (HIPAA, VPC, tool governance). Neither has the other's unique capabilities yet.

### Context Management Divergence
The two platforms are taking different approaches to context management in long-running agent sessions:
- **Anthropic**: Compaction API (server-side summarization), context editing (selective removal), effort levels (low/medium/high/max)
- **Google**: Context compression services (Gemini CLI v0.38.0-preview), context circulation (API-level multi-step context preservation), 4-tier configurable context layers (Static/Turn/User/Cache)
- **Observation**: Google's approach is more granular (4 tiers + compression + circulation), while Anthropic's is simpler (compact or edit). Both acknowledge this as the critical production pain point.
