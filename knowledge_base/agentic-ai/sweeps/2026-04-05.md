## Cross-Cutting Analysis
- **Subscription vs API tension**: The OpenClaw cutoff reveals that flat-rate subscriptions fundamentally cannot support agentic workloads. Google's parallel move (free-tier restricted to Flash-only) confirms both vendors are repricing for agentic usage patterns. Developer backlash is significant — the OpenClaw creator joined OpenAI within weeks of the announcement, and the broader pattern (Google, Microsoft, OpenAI all restricting third-party access) suggests an industry-wide shift toward platform-first monetization.
- **Agent CLI convergence deepens**: Gemini CLI v0.36.0 stable now has git worktree support — the exact same pattern Claude Code uses for agent isolation. Both CLIs now share: markdown-defined agents, project config files, Agent Skills, MCP integration, subagent isolation, tool allowlists, and now git worktrees. Gemini CLI's experimental memory manager agent (replacing save_memory tool) also parallels Conway's persistent memory. This convergence validates our pipeline's SKILL.md format as industry-aligned.
- **Agent Payment Protocol War**: Four competing protocols (AP2, Visa TAP, x402, PayPal Agent Ready) launched within 90 days. Rather than winner-take-all, they're converging into a layered architecture: trust (TAP/AP2) → intent (AP2 mandates) → settlement (x402 crypto, traditional fiat rails). $11.79B market projection for 2026. Anthropic has MCP but no payment protocol — this is now a three-way gap (no A2A, no AP2, no payment layer).
- **Supply chain security**: The LiteLLM compromise affecting ADK (3 patches in 4 days) is the first visible supply chain security incident in the agent framework ecosystem. Combined with MCP tool poisoning, agent security is moving from theoretical to actively exploited.
- **MCP ecosystem maturing**: Registry reached 6,400+ servers. New verticals emerging — Fingerprint's fraud prevention MCP server shows MCP expanding beyond developer tools into fintech/security. Pinterest's two-layer auth model remains the best enterprise pattern. Both tool poisoning and Computer Use prompt injection show the security surface expanding faster than defenses.
- **Persistent agents emerging**: Conway's webhook-driven wake model + extension system signals that the industry is moving from session-based to event-driven agent architectures. This aligns with our cron-based steward model but suggests webhook-driven activation may be the next evolution.
- **Scientific reasoning race**: Gemini 3 Deep Think's expansion into chemistry/physics (gold medal-level IPhO/IChO) opens a new frontier. Anthropic has no specialized scientific reasoning mode. This could unlock agent use cases in lab automation, research workflows, and engineering analysis.
- **Edge AI race**: Gemma 4 AICore with hardware acceleration (Google/MediaTek/Qualcomm) vs no Anthropic equivalent for on-device models. Google has a massive lead in edge agent deployment.

## Implications for Our Pipeline
- **Conway watch**: If Conway ships with `.cnw.zip` extensions, our Skills could potentially be packaged as Conway extensions — worth prototyping in Phase 5
- **MCP tool poisoning**: Our pipeline ingests MCP tools — we should validate tool descriptions from untrusted servers before injecting them into agent context
- **Agent SDK context usage**: The `get_context_usage()` API could improve our long-running steward agents' context management — currently they rely on model-side compaction
- **Agent SDK MCP fix**: The v0.1.55 fix for silently truncated large MCP tool results is directly relevant — any SDK-based agents using MCP servers that return >50K chars should upgrade immediately
- **OpenClaw cutoff**: Confirms API-first pricing for agent workloads — our cost modeling should assume API pricing, not subscription. Developer backlash suggests market opportunity for cost-efficient agent frameworks.
- **Payment protocols for Phase 7**: All four agent payment protocols (AP2, TAP, x402, PayPal) should be tracked. AP2/x402 are most relevant for A2A/MCP ecosystem; PayPal Agent Ready's protocol-agnostic approach may be the best integration target.
- **Gemini CLI convergence**: Git worktree support in Gemini CLI v0.36.0 further validates our SKILL.md-based approach. Cross-tool compatibility testing (our Skills in `.gemini/skills/`) now has a concrete target.
- **Scientific reasoning**: Gemini 3 Deep Think's science capabilities could be valuable for BSP Knowledge steward's physics-oriented ARM TRM analysis — worth evaluating.
