### Agent SDK

- **Python v0.1.66 (Apr 23)**: bundled CLI bump to v2.1.119 only.
- **Python v0.1.67 (Apr 25 00:27 UTC)**: trio compatibility restored — fixes 16-version regression (`RuntimeError: no running event loop` since v0.1.51). Implementation uses sniffio-based runtime dispatch (`asyncio.Task` vs `trio.lowlevel.spawn_system_task`). New explicit dependency `sniffio>=1.0.0`. Bundled CC v2.1.120 (the silent release).
- **Python v0.1.68 (Apr 25 02:07 UTC)**: bundled CLI bump only (CC v2.1.119, regressing from v0.1.67's v2.1.120 bundle — note this).
- **TypeScript v0.2.119 (Apr 23 23:24 UTC, paired with CC v2.1.119)**: three changes: (a) `excludeDynamicSections` keeps static auto-memory in cacheable system-prompt block (improves prompt-cache hit rate), (b) long-running SDK sessions reconnect claude.ai-proxied MCP servers after transport-stream abort, (c) `SessionStore.append()` retries up to 3 times with backoff before drop + `mirror_error`.
- **Subagent resumability now documented**: `agentId` returned in Agent tool result; parent can `resume: sessionId` and name `agentId` to continue. Subagent transcripts persist in separate files, are unaffected by main-conversation compaction, default 30-day cleanup. **First long-running multi-agent pattern available without external state stores**. Source: https://code.claude.com/docs/en/agent-sdk/subagents
- **Task→Agent rename is partial**: SDK still emits `"Task"` in `system:init` and `result.permission_denials[].tool_name`. Detection logic must match both names.
