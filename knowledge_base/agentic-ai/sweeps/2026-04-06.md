## Cross-Cutting Analysis

### Industry Stabilization Phase (April 5-6)
Both Anthropic and Google are in a clear stabilization/consolidation phase after the intense March/early April release burst. This is consistent with:
- Pre-conference preparation (Google I/O 2026, May 19-20)
- Post-launch quality hardening (Claude Code v2.1.92, Gemini CLI v0.36.0, Gemma 4)
- Security incident response (MCP CVEs, LiteLLM compromise, tool poisoning)

### Harness Engineering Emerges as Discipline
Anthropic's three-agent harness blog has catalyzed a new named practice: "harness engineering" — designing multi-agent architectures where each component encodes assumptions about model limitations. This sits alongside prompt engineering (single-turn) and context engineering (multi-turn) as the third pillar of applied AI engineering. The key insight: as models improve, harness components should be progressively simplified, not accumulated.

### MCP Security at Inflection Point
The OWASP MCP Top 10 + quantified metrics (84.2% attack success rate, 34/100 avg security score) signal MCP is at a critical security inflection point. The protocol is widely adopted (10K+ servers) but poorly secured. This creates both risk and opportunity:
- **Risk**: Enterprises may slow MCP adoption pending security hardening
- **Opportunity**: First-movers on MCP security tooling (mcp-scan, tool pinning, content validation) will capture enterprise trust

### Architectural Convergence Confirmed
Gemini CLI v0.36.0 and Claude Code v2.1.92 continue converging:
- Both: markdown agent definitions, project config files, MCP integration, git worktree isolation, skills/commands system
- Both: memory management (auto memory vs experimental memory manager agent)
- Divergence: Claude Code has deeper hook system (15+ events); Gemini CLI has native A2A support
