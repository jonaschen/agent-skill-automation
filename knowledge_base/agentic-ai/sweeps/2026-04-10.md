## Cross-Cutting Analysis
- **Protocol convergence accelerating**: A2A's triple hyperscaler integration (Azure + AWS + Google) and 150+ organizations makes it the de facto standard for enterprise inter-agent communication. MCP remains dominant for agent-to-tool integration. AP2 (60+ orgs) fills the payments layer. The A2A+MCP+AP2 stack is now the canonical reference architecture.
- **Enterprise agent stacks crystallizing**: Anthropic now offers Claude Code + Cowork (consumer/enterprise) + Managed Agents (cloud API) + Agent SDK (custom). Google offers Gemini CLI + ADK + Agent Engine + Gemini Enterprise. Both stacks are production-ready.
- **Memory leak as systemic issue**: Claude Code's MCP HTTP/SSE leak (~50 MB/hr) and Gemma 4's SWAKVPool memory leak both surfaced this week. Long-running agent sessions are stress-testing memory management across the ecosystem.
- **Safety gates converging**: Gemini CLI nightly adds user confirmation for skill activation (mirroring Claude Code's permission system). Both CLI agents now require explicit approval before activating skills — reducing unintended skill execution risk.
- **Pre-I/O freeze deepening**: ADK day 8 freeze (longest ever), Vertex AI Agent Builder 51-day silence, and Gemini CLI stable freeze all point to a major Google announcement at I/O (May 19-20, 39 days).

## Implications for Our Pipeline
- **MCP memory leak fix is critical for steward agents**: Our nightly agents run 30-60 min sessions. The 50 MB/hr leak was potentially causing OOM in longer sessions. Update Claude Code to v2.1.97 immediately.
- **Cowork enterprise GA + Managed Agents = complete Anthropic enterprise stack**: For Phase 7 AaaS, the Managed Agents pricing ($0.08/session-hour + tokens) provides a concrete cost model for hosted skill execution.
- **A2A triple hyperscaler integration validates Phase 5 design**: Our Phase 5 multi-agent topology should use A2A as the inter-agent communication protocol — it's now production-proven across all three cloud platforms.
- **Haiku 3 retirement in 9 days**: Verify our deprecation guard catches this. Run `eval/model_deprecation_check.sh` to confirm.
- **EAGLE3 approach for Phase 6**: The 1.72x speedup for Gemma 4 31B via speculative decoding could be adopted for on-device skill inference optimization.
- **Effort level default raised to high**: Our steward agents running on API keys now default to high effort — may increase token costs but improve output quality. Consider explicit `effort: medium` in agent configs to control costs.
