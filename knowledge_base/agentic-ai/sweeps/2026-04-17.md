### Multi-Agent Patterns
- **Three-Agent Harness formalized (Apr 4 blog "Harness Design for Long-Running Apps")**: (1) Planning Agent — task decomposition, strategy, success criteria. (2) Generation Agent — code/design/implementation. (3) Evaluation Agent — independent grading (separation avoids self-evaluation bias). **Structured handoff artifacts** replace shared rolling context; explicitly contrasted with compaction ("can make models overly cautious about approaching token limits"). Concrete implementation: **Playwright MCP evaluator** scoring 4 axes (design quality, originality, craft, functionality). 5–15 refinement iterations per run, sometimes ~4 hours of autonomous operation. Anthropic explicit: "separation of evaluation from generation is a strong lever." ([anthropic.com](https://www.anthropic.com/engineering/harness-design-long-running-apps), [infoq.com](https://www.infoq.com/news/2026/04/anthropic-three-agent-harness-ai/))
- **Multi-Agent Research System blog** (orchestrator-worker pattern): 3–5 parallel specialists + 3+ tool calls each = 90% time reduction on complex queries. Synchronous cohort + external memory. Tool description quality → 40% task completion time reduction for future agents. Rainbow deployments for long-running processes.
- **2026 Agentic Coding Trends Report**: 8 trends — multi-agent coordination now mainstream (not experimental). Case studies: Rakuten, CRED, TELUS, Zapier.

## Cross-Cutting Analysis

### Three converging architectural patterns (Anthropic side)
Anthropic has now published, within two weeks, three distinct multi-agent architectures that converge on the same principle (separation of concerns across agent roles):
1. **Three-Agent Harness** (long-running app generation) — Planning / Generation / Evaluation, sequential with structured handoffs.
2. **Multi-Agent Research System** (research agents) — orchestrator + 3–5 parallel specialists with external memory.
3. **Advisor Tool** (inline model pairing) — executor model consults advisor mid-generation as a server-side sub-inference.

Our pipeline already maps 1:1 onto pattern (1): meta-agent-factory (Generation) ↔ skill-quality-validator (Evaluation) ↔ autoresearch-optimizer (Planning/replanning). **Our Phase 5 TCI router should treat these three patterns as three canonical topology choices**: sequential harness (high-depth tasks), parallel orchestrator (high-branching research), inline advisor (low-coupling tasks).

### Pricing strategy divergence
- **Anthropic**: Opus 4.7 pricing held flat at $5/$25 MTok despite step-change capability — absorbing capability gains to hold price/quality lead.
- **Google**: Gemini API changelog 2 days silent; Vertex AI Agent Builder 58 days silent. Not a pricing signal, but a **platform freeze signal** — holding for I/O relaunch.
- **Convergent conclusion**: Both vendors are currently withholding commercial surface changes for strategic reasons (Anthropic: price competition; Google: product relaunch). Our pipeline can plan 4-week roadmap with high confidence that no pricing structure shifts beneath us.
