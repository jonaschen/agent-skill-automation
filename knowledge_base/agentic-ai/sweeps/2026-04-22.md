## Anthropic Updates

### Claude Code
- **v2.1.117** (Apr 22) — Major release ending 2-day silence since v2.1.116 (Apr 20):
  - **Opus 4.7 context window fix**: CC computed against 200K instead of 1M, causing premature autocompacting — fixed. Inflated `/context` percentages resolved.
  - **Forked subagents**: `CLAUDE_CODE_FORK_SUBAGENT=1` enables on external builds — democratizes isolation pattern.
  - **Agent frontmatter `mcpServers`**: Declarative MCP dependency loading for agents via `--agent`.
  - **Native bfs/ugrep**: Replaces Glob/Grep tools on macOS/Linux native builds — faster searches without tool round-trip.
  - **Default effort `high`**: For Pro/Max on Opus 4.6/Sonnet 4.6 (was `medium`).
  - **OTEL enhancements**: `effort` attribute on cost/token events; `command_name`/`command_source` for slash commands.
  - **Advisor Tool**: Stability fixes (no more "could not be processed" stuck state), experimental labeling.
  - **Managed-settings**: `blockedMarketplaces`/`strictKnownMarketplaces` enforced.
  - **Plugin improvements**: Missing dependency auto-install, marketplace dependency resolution.
  - **Concurrent MCP**: Now default when both local and claude.ai MCP servers configured.
  - **Bug fixes (14+)**: OAuth token refresh on 401, WebFetch hang on large HTML, proxy 204 crash, NO_PROXY under Bun, idle re-render loop on Linux, subagent malware false positive for different-model subagents, Bedrock/Opus 4.7 with thinking disabled 400 error, MCP elicitation auto-cancel in print/SDK mode, `/resume` stale session summarization.
