## Cross-Cutting Analysis

- **CLI convergence deepens**: Gemini CLI v0.38.0's context compression service vs Claude Code's conversation compaction — both solving the long-session context problem but architecturally differently (Google: explicit service; Anthropic: implicit system-level). The `/memory inbox` in v0.39.0-preview.0 makes the skill lifecycle more structured than Claude Code's explicit memory commands.
- **TTS as agent capability**: Gemini 3.1 Flash TTS with audio tags represents a significant gap vs Anthropic — Anthropic has no equivalent text-to-speech model. Agentic voice applications (phone call agents, voice assistants, audio content generation) are now first-class on Gemini but require third-party TTS on Claude.
- **Open stack completion**: ADK v1.30.0 + Gemma 4 = first time Google offers a fully open-weight, open-framework, managed-runtime agent stack. Anthropic's equivalent would require Claude API (proprietary), Agent SDK (open), and third-party hosting — no fully open option.
- **I/O pre-announcement window**: ~17 days to pre-I/O window. Items to watch: ADK v2.0 beta/GA, Gemini 4 reveal, AI Applications platform refresh, Android XR/Samsung glasses formal announcement, Mariner Studio preview.
