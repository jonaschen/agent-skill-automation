## Cross-Cutting Analysis

### Convergence: Brain-Hands Architecture
Both vendors are converging on a stateless-orchestrator + ephemeral-execution pattern:
- **Anthropic Managed Agents**: harness (brain) invokes sandbox (hands) via `execute(name, input) → string`
- **ADK v2.0-alpha.3**: workflow graph orchestration with lazy scan dedup and resume
- **Implication**: Our Phase 5 topology should adopt this pattern — stateless router + ephemeral agent execution.

### Divergence: Pre-I/O Freeze Depth
Google's entire AI developer stack is in the deepest observed freeze (10+ days API silence, 53-day Agent Builder gap). Anthropic continues shipping (v2.1.101 on Apr 10). This suggests Google is preparing a massive I/O reveal while Anthropic executes incrementally.

### Security Hardening Cycle
Both vendors simultaneously hardening:
- **Anthropic**: PID namespace isolation, `permissions.deny` override fix, `which` command injection fix
- **Google ADK**: BashTool metacharacter blocking, subprocess resource limits, shell security
- Pattern: the same attack vectors (command injection, permission escalation, sandbox escape) are being addressed in parallel.

### Credential Isolation Convergence
- **Anthropic Managed Agents**: vault-based MCP proxies — credentials never reach sandboxes
- **AWS MCP**: IAM context keys differentiate agent vs human actions
- **Our pipeline**: currently uses environment variables — less secure than either pattern.

### Reactive vs Scheduled Agents
- MCP Triggers & Events working group (led by new maintainer Liguori) could enable **reactive agent activation** — agents triggered by events rather than cron schedules. If this lands in the June spec, it could fundamentally change our steward agent scheduling model.
