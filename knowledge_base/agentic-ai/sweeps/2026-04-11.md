### Tool Use & Function Calling — ADVISOR TOOL LAUNCHED
- **Advisor Tool** (`advisor_20260301`, public beta, April 9): executor model (Haiku/Sonnet/Opus) consults Opus advisor mid-generation as server-side sub-inference within single `/v1/messages` request. Advisor reads full transcript, returns 400-700 tokens of guidance. Separate `usage.iterations[]` billing. Caching available (`5m` or `1h` TTL). Max_uses parameter caps calls per request. Valid pairs: Haiku→Opus, Sonnet→Opus, Opus→Opus.
- **Benchmark results**: Sonnet+Opus advisor: 2.7pp improvement on SWE-bench Multilingual, 11.9% cost reduction per agentic task. Haiku+Opus advisor: 41.2% on BrowseComp (vs 19.7% solo), 85% cheaper than Sonnet solo.
- **Recommended pattern**: call advisor before substantive work, when stuck, and before declaring done.
- Source: https://platform.claude.com/docs/en/agents-and-tools/tool-use/advisor-tool, https://claude.com/blog/the-advisor-strategy

## Cross-Cutting Analysis
- **Advisor Tool vs ADK EnvironmentToolset**: Both are "capability amplifiers" released the same day (April 9). Anthropic amplifies model quality (advisor gives cheaper models better reasoning), Google amplifies model reach (EnvironmentToolset gives ADK agents filesystem/shell access). Different strategies: Anthropic invests in inference-level innovation; Google invests in tool-level infrastructure.
- **ADK v2.0 workflow graph converges with Vercel WDK**: The lazy scan dedup on resume in ADK v2.0.0-alpha.3 mirrors Vercel Workflow DevKit's deterministic replay — both solve the same problem (don't re-run completed work after restart) from different directions.
- **MTP omission creates strategic asymmetry**: Google maintains inference speed advantage over open-source Gemma 4 deployments. EAGLE3 (1.72x) partially compensates but requires custom training. TRL v1.0 provides alignment training capability that mitigates quality concerns but not speed.
- **Pre-I/O freeze confirmed on Google side**: ADK's 8-day freeze broken with v1.29.0 + v2.0.0-alpha.3, Vertex AI at 52 days silence, Mariner/Astra quiet. Google is staging for I/O (May 19-20). Expect ADK v2.0 beta, A2A ecosystem tooling, Android XR glasses reveal, potentially Gemini 4 preview.
- **Anthropic custom chip exploration**: $30B ARR + 3.5 GW TPU deal + custom chip exploration signals Anthropic preparing for massive scale-up. If custom chips materialize, it reduces Google dependency — significant for the competitive landscape.

## Implications for Our Pipeline
- **Phase 3 (optimizer)**: Advisor pattern could reduce optimizer iteration cost — use Sonnet+Opus advisor instead of pure Opus for autoresearch iterations. 11.9% cost reduction confirmed.
- **Phase 4 (closed loop)**: Agent SDK v0.1.57's cross-user prompt caching directly benefits our daily agent fleet if we adopt Agent SDK — shared system prompts across stewards would reduce input token costs 40-60%.
- **Phase 5 (topology)**: Advisor tool creates a new topology option: advisor-pattern for low-coupling tasks (cheaper) and full agent teams for high-coupling tasks. The `usage.iterations[]` billing provides per-agent cost attribution model.
- **Phase 6 (edge)**: Gemma 4 MTP omission means self-hosted inference is slower than Google's internal serving. TRL v1.0 provides alignment training path. EAGLE3 partially compensates. ADK EnvironmentToolset provides a reference for agent tool design.
- **Phase 7 (AaaS)**: Advisor tool's `usage.iterations[]` billing model is a template for per-component billing in outcome-based pricing.
