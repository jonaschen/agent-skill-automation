# Agentic AI Sweep Report — 2026-04-02

## Executive Summary

- **MCP and A2A are converging under the Linux Foundation's Agentic AI Foundation (AAIF)** — 146 members, both protocols now governed jointly. The emerging consensus is a three-layer stack: WebMCP (web), MCP (tools), A2A (agent coordination).
- **Anthropic shipped agent teams in Claude Code** (Feb 2026) alongside Opus 4.6 — multi-agent coding is now a first-class feature with Git worktree isolation and task board coordination.
- **Google ADK hit 1.0 across 4 languages** (Python, Go, Java, TypeScript) with native MCP + A2A support, making it the most polyglot agent framework.
- **SWE-bench Verified scores are compressed** — top 5 models within 2 points (Claude Opus 4.5: 80.9%, Gemini 3.1 Pro: 80.6%, GPT-5.2: 80.0%). Cost efficiency now a key differentiator.
- **Gemma 3n enables on-device agents** at 2-3GB memory with function calling — directly relevant to our Phase 6 edge AI plans.

## Cross-Cutting Analysis

### MCP vs A2A: Complementary, Not Competing
- MCP handles agent-to-tool (97M monthly SDK downloads); A2A handles agent-to-agent (150+ orgs)
- Both now under Linux Foundation's AAIF (146 members including AWS, Anthropic, Google, Microsoft, OpenAI)
- Emerging three-layer stack: WebMCP (web access) + MCP (tool access) + A2A (agent coordination)
- Additional protocols tracked: UTCP, ANP, NLIP, A2UI, AG-UI, UCP, AP2

### Benchmarks: Top is Compressed
- SWE-bench Verified: Claude Opus 4.5 (80.9%), Opus 4.6 (80.8%), Gemini 3.1 Pro (80.6%), GPT-5.2 (80.0%)
- GAIA approaching saturation at 90% (human baseline 92%)
- WebArena improved from 14% to ~60% in 2 years; 18-point gap to human (78%) remains
- Cost efficiency is the new differentiator: Gemini 3 Flash at $0.356/instance vs Claude Opus 4.5 at $0.754/instance

## Implications for Our Pipeline

1. **Phase 3 (Optimizer)**: Programmatic tool calling GA eliminates N round-trips — our `run_eval_async.py` could batch tool calls in a single code execution container. `strict: true` should be adopted for all eval tool definitions
2. **Phase 4 (Closed Loop)**: `defer` hooks enable headless CI/CD agents that pause at permission boundaries and resume — directly maps to our deployment gate. `--bare` mode is ideal for scripted eval invocations. Agent SDK `startup()` pre-warm (20x faster first query) critical for eval loop latency
3. **Phase 4 (Closed Loop)**: Conditional hooks (`if: Bash(git push *)`) reduce overhead — our `pre-deploy.sh` should adopt this. HTTP hooks enable cloud-native deployment notifications
4. **Phase 5 (Multi-Agent Topology)**: Anthropic's agent teams architecture (file-locking task claims, quality gate hooks, adversarial debate pattern, plan approval mode) is a production-validated reference for our TCI-based routing. The 15x token cost of multi-agent is an important cost model input
5. **Phase 5 (Multi-Agent Topology)**: Agent SDK subagent introspection (`listSubagents()`, `getSubagentMessages()`) enables post-hoc analysis of multi-agent runs — valuable for our topology watchdog
6. **Phase 6 (Edge AI)**: Gemma 3n (2-3GB memory, function calling) and FunctionGemma (270M) are prime candidates for on-device agent components — ONNX/GGUF packaging aligns directly
7. **Safety/Quality**: The enterprise security gap (81% deployed, 14.4% approved) is an opportunity — our CI/CD gate and validator pipeline addresses exactly this need
8. **Model Migration**: `output_format` → `output_config.format` breaking change needs attention if we use structured outputs. Prefill removal on Opus 4.6 may affect agent workflows
9. **MCP Auth**: OAuth 2.1 + CIMD auth framework maturity means our MCP-integrated agents should plan for auth flows in production deployments
