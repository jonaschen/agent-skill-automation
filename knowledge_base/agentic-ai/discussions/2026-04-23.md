### INNOVATOR

**Proposal 1.1: Phase 5 Hybrid Routing — Combining Static TCI with Dynamic Graph Edges (S2)**

Today's analysis correctly identifies that ADK v2.0's graph routing and our TCI routing solve different problems (resource allocation vs. workflow structure). But the analysis stops short of the design synthesis: we should **combine both**. Our Phase 5 could use TCI at dispatch time to decide parallelism, then within each track, use graph-style conditional edges for dynamic branching.

Concretely: the `closed_loop.sh` state machine already implements conditional branching imperatively — `if eval_score >= 0.95 then skip_optimize; elif optimizer_retries > 3 then report_failure`. This is a workflow graph expressed as bash `if/else`. ADK v2.0 shows the value of making this declarative: each state becomes a node, edges carry conditions, and the orchestrator runtime handles dispatch.

Concrete proposal: Add a "Hybrid Routing Architecture" subsection to the Phase 5 design. Contents: (1) TCI at dispatch (existing design, unchanged), (2) within-track declarative workflow graph for pipeline stages (GENERATE → VALIDATE → conditional branch on score → OPTIMIZE or DEPLOY), (3) note that this is an abstraction over the existing `closed_loop.sh` state machine, not a new capability. The graph visualization ADK v2.0 ships would provide Jonas operational visibility as a Phase 7 observability item.

Implementation: ~15 minutes. Documentation in the Phase 5 design area. No code changes. Feeds Design Freeze (May 22-26).

The agent-centric vs. workflow-centric framing is novel enough to anchor a paper section.

Implementation: ~20 minutes. Knowledge base content only.
