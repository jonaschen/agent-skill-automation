## 2. Strategic Priority Alignment

### S1: Automatic Agent/Skill Improvement

**Today's contribution: OTEL tracing unblocks automated behavioral analysis (HIGH impact)**

The single most impactful finding for S1 is that OTEL tracing works with `claude -p` via 6 env vars — no SDK migration needed. This transforms from "months of work" to "days of work." With traces:

- Every tool call, API request, and subagent invocation becomes a structured span
- Token/cost counters as metrics enable automated cost anomaly detection
- Trace trees enable automated analysis of agent decision patterns
- `session.id` correlation supports cross-session behavioral comparison

**S1 gap narrowing**: Before today, the S1 loop was: researcher tracks → research-lead prioritizes → factory implements. After today, the infrastructure for a tighter loop exists: OTEL traces → automated analysis → optimization triggers. This is the "capability diff" mechanism S1 asks for — but at the agent behavior level, not the platform capability level.

**Remaining S1 gap**: No automated system that reads OTEL traces and generates agent improvement suggestions. This is a Phase 5 deliverable, but the observability prerequisite is now unblocked.

### S2: Multi-Agent Orchestration

**Today's contribution: Five-pattern taxonomy + ADK Session Rewind (MEDIUM impact)**

Two findings advance S2:

1. **Five-pattern multi-agent taxonomy** (Teams, Subagents, Three-Agent Harness, Orchestrator-Worker, + Message Bus + Shared State): Expands the design space for Phase 5 TCI routing. However, the directive correctly defers implementation until the 50-task TCI benchmark produces distribution data.

2. **ADK Session Rewind**: Introduces session-level checkpointing as an orchestration primitive. Our current pipeline uses coarse-grained git-based rollback. Session Rewind maps to a capability our optimizer needs: mid-session undo without losing context. Cross-pollination path: Agent SDK session management (when we migrate from `claude -p` in Phase 5).

**S2 gap remaining**: No original orchestration experiments designed or run. The findings are analytical (tracking vendor patterns) rather than experimental. The directive's "output throttling" guidance correctly limits proposals, but S2's meta-goal (original research + experimentation) needs a different cadence — perhaps a dedicated experiment design session rather than sweep-cycle proposals.

### S3: Platform Generalization (Cross-Platform Agent Portability)

**Today's contribution: Routines vs. cron comparison + agent ecosystem convergence (LOW impact)**

Two minor S3 data points:

1. **Routines comparison** confirms our cron infrastructure is superior for multi-agent scheduling. Routines' API/GitHub triggers are interesting for single-agent HITL workflows but can't replace our pipeline.

2. **Ecosystem convergence**: Both vendors now have session rewind (ADK explicit, Claude git-based), sandboxed execution (ADK AgentEngineSandbox, Anthropic Managed Agents containers), computer use (both in preview), and multi-agent patterns (ADK graph workflows, Anthropic four patterns). The convergence suggests a portable agent definition format is architecturally feasible — the underlying capabilities are aligning.

**S3 gap remaining**: No concrete analysis of Gemini CLI's agent definition format vs. SKILL.md. This is the single most important S3 research task and hasn't been attempted yet. The pre-I/O freeze makes it a good time to analyze the stable Gemini CLI v0.38.x format before I/O changes it.
