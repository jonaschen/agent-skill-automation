### 1.4 MCP Memory Leak — FLEET OPERATIONAL IMPACT (P1 — Operational)

**Finding**: Claude Code v2.1.97 fixes an MCP HTTP/SSE memory leak accumulating ~50 MB/hr from unreleased buffers on reconnect.

**Impact on our pipeline**:

- **Direct exposure**: Our 8 daily agents run 30-60 min sessions. At 50 MB/hr, a 60-min session leaks ~50 MB. This is unlikely to cause OOM on modern systems with >8GB RAM, but:
  - The leak is cumulative across reconnects. If an MCP server drops and reconnects multiple times during a session, the leak accelerates.
  - Our factory-steward runs 3 sessions/day. If any session has MCP connectivity issues, memory accumulates across the cron runner process.
  - The researcher's sweep sessions can run longer (60+ min) with active WebSearch/WebFetch, which may use MCP-like HTTP connections internally.

- **Severity for our fleet**: MEDIUM. Our sessions are short enough that 50 MB/hr won't cause crashes. But the fix should be applied to prevent accumulation in edge cases.

**Current state**: Need to verify what Claude Code version our fleet is running.

**Recommendation**: P1 — Update Claude Code to v2.1.97. This also picks up the 429 retry fix (which was burning all retry attempts in ~13s) and the permission hardening fixes. The 429 fix is arguably more impactful for our fleet, since our eval scripts hit rate limits regularly.

This is complementary — our hook provides client-side monitoring, the server provides server-side enforcement.
- **`additionalDirectories` mid-session**: Our steward agents operate on target repos specified in their configs. If `additionalDirectories` wasn't applying mid-session, agents may have had narrower filesystem access than intended.

**Gap**: No audit of our settings.json for prototype-collision-vulnerable keys.

**Recommendation**: P1 — Quick audit: grep our `.claude/` config files for keys that match JS prototype property names (`toString`, `valueOf`, `hasOwnProperty`, `constructor`, `__proto__`). Likely clean, but 30-second verification.

1. **Anthropic**: Claude Code → Cowork → Managed Agents → Agent SDK (closed ecosystem, premium positioning, $30B ARR)
