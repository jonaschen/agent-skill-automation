## 2. Cross-Pollination Opportunities

### 2.1 ADK Events/Trace View → Our OTEL Implementation

**ADK context**: v1.31.0 (Apr 17) shipped Events and Trace View — replaced execution logs with structured tracing (spans, events) and Graph View (visual canvas for agent architectures).

**Our context**: Phase 5 OTEL tracing is designed but not implemented. The evening analysis (2026-04-18) confirmed OTEL tracing does NOT require SDK migration — env vars on `claude -p` invocations are sufficient.

**Cross-pollination**: ADK's Events/Trace View demonstrates the concrete value of structured tracing for multi-agent debugging. Our OTEL implementation should target the same three outputs:
1. **Span-level traces** (already planned via `CLAUDE_CODE_ENABLE_TELEMETRY=1`)
2. **Event view** (map to our session_log.sh JSONL events — SESSION_START, TASK_COMPLETE, etc.)
3. **Graph view** (deferred — Phase 5 topology visualization, not Phase 4 scope)

The key insight from ADK is that trace + event + graph are three independent views of the same execution. Our session_log.sh already provides events; OTEL env vars provide traces. These can be unified without a shared database — each view reads its own format.

### 2.2 ADK Session Rewind → Optimizer Branch Search

**ADK context**: v1.31.0 Session Rewind allows rewinding a session to before a previous invocation. Users can "undo" the last agent action and retry with different parameters.

**Our context**: The autoresearch-optimizer uses parallel branch search (A/B/C/D) with file-level rollback between iterations. The Phase 5 design note (2026-04-18) already flagged Session Rewind as potentially enabling "rewind-to-checkpoint" behavior for the optimizer.

**Cross-pollination**: Session Rewind solves a different problem than file rollback. File rollback reverts SKILL.md changes. Session Rewind would revert the agent's *understanding* of why a change failed — preserving in-session learning across rewinds. This is more valuable for the optimizer than file rollback because the optimizer's key asset is its accumulated understanding of failure patterns. However, Session Rewind requires Agent SDK sessions (Phase 5.3.3 migration), so this remains a Phase 5 consideration.

### 2.3 Dual Observability Pattern: OTEL + Session Storage

**Anthropic context**: Agent SDK TS v0.2.113 shipped Session Storage Alpha. Combined with OTEL traces, this creates dual observability: OTEL for structured span-level metrics, Session Storage for full conversation replay.

**Google context**: ADK v1.31.0 Events/Trace View provides structured tracing. ADK sessions are already file-backed (session state directory).

**Cross-pollination**: Both vendors have converged on the same insight — structured traces alone are insufficient for debugging agentic systems. You also need full conversation replay to understand *why* an agent made a decision, not just *what* it did. Our pipeline has a partial substitute: `logs/sessions/*.jsonl` (events) + `logs/*.log` (raw output). Phase 5 should unify these into the dual-channel pattern.
